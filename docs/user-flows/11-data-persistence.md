# Where Your Data Is Stored

ReqResFlow saves all your data as files on your computer. Everything is saved automatically — you don't need to manually export or back up.

## What Gets Saved

1. **Collections** — all your saved collections and their requests.
2. **Environments** — all your environments and their variables.
3. **History** — the last 100 requests you've sent.
4. **Session** — which tabs are open, which tab is active, which environment is selected.
5. **Flows** — all your multi-step request sequences.

## When Does Saving Happen?

| Data          | Saved when...                                                                                  |
| ------------- | ---------------------------------------------------------------------------------------------- |
| Collections   | You create, delete, rename, or update a collection or request.                                 |
| Environments  | You edit variables, or when captures update a variable after sending a request.                 |
| History       | You send a request or clear history.                                                           |
| Session       | You open/close/switch tabs, edit a request, or change environments.                            |
| Flows         | You create, delete, rename, or save a flow.                                                    |

## What Happens If the App Crashes?

- Because everything is saved continuously, you'll lose almost nothing.
- When you reopen the app, your tabs, collections, environments, history, and flows will all be restored to their last saved state.

## What Happens If a Save File Gets Corrupted?

- The app handles this gracefully.
- If a file can't be read, the app starts with empty data for that category (e.g., empty collections list, no history) rather than crashing.
